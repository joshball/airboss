# User Stories: Discovery

## Onboarding

- As a new learner, I want to tell the system about my background so the training is tailored to where I actually am, not a generic baseline.
- As an experienced CFI, I want the system to know I have 10+ years so it doesn't start me on the absolute basics.
- As a learner with TAA experience, I want to flag that so I'm not shown intro-level glass cockpit content.

## Self-assessment

- As a learner, I want to rate my confidence in each training domain so the course adapts to my real weak areas, not just a random starting point.
- As a learner who is unsure about a domain, I want a "Not sure" option so I'm not forced to guess at a rating I can't honestly give.
- As a learner, I want the domain names explained in plain terms so I know what I'm actually rating.

## Goals

- As a learner with specific concerns (e.g., TSA compliance), I want to flag those so the system emphasizes them early.
- As a learner who just wants to get started, I want to skip the goals step without being penalized.

## Flow

- As a learner, I want to complete discovery in one sitting so I don't lose my progress if I close the tab.
- As a learner who skipped discovery, I want to come back and complete it later from settings so I haven't permanently locked myself out of personalization.
- As a learner who already completed discovery, I want the system to not show me the intake again unless I explicitly ask to redo it.

## Engine connection

- As a learner, I want my background and self-assessment to visibly influence the scenarios I see first, so I feel like the system actually listened.
- As an operator, I want learner profile data stored so we can analyze whether self-assessment correlates with actual performance.
