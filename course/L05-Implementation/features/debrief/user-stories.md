# User Stories: Debrief

## Outcome

- As a learner, I want to immediately know if my student was safe after the scenario so I understand the stakes of my decisions.
- As a learner, I want the outcome framed in terms of the situation ("Student at risk") rather than my performance ("You failed") so I focus on understanding, not defensiveness.

## Analysis

- As a learner, I want to see a tick-by-tick breakdown of what happened so I can trace exactly where the situation evolved.
- As a learner, I want to see what a better intervention would have been (when I chose wrong) so I have a concrete alternative to take into my next attempt.
- As a learner, I want to see when I intervened at the right level so I know what I got right, not just what I missed.
- As a learner, I want to see the 2-3 biggest contributing factors to the outcome so I know where to focus my attention for the retry.

## Score

- As a learner, I want my score shown as four labeled dimensions (not a single number) so I understand what specifically I'm good or weak at.
- As a learner, I do NOT want a letter grade or pass/fail score so the debrief doesn't feel like a report card.

## Replay

- As a learner who got an unsafe outcome, I want a prominent "Try Again" button so retrying feels like the obvious next step, not a consolation prize.
- As a learner, I want a "Replay" option to watch the annotated version of my run so I can see the chain without the time pressure.

## FAA / Compliance

- As an operator, I want to know which FAA topics and competencies were exercised in each run so we can demonstrate topic coverage to the FAA.
- As a learner, I want to see the FAA topics and competencies tagged on my debrief so I know this training is purposeful, not arbitrary.
