# User Stories: Scenario Player

## Briefing

- As a learner, I want to read a briefing before the scenario starts so I understand the situation and who my student is.
- As a learner, I want to see which competencies the scenario targets so I know what I'm being evaluated on.
- As a learner, I want a clear "start" moment (Call the Ball) so I know when the clock starts and I'm committed.
- As a learner, I want to know approximately how long the scenario takes so I can decide if I have time to start now.

## Playing

- As a learner, I want to see the evolving situation as the scenario progresses so I can build a mental picture of what's happening.
- As a learner, I want to see what my student is saying and doing so I can diagnose their state.
- As a learner, I want five distinct intervention options so I can apply the right level of response (not just "right/wrong").
- As a learner, I want time pressure during critical moments so the intervention feels real, not academic.
- As a learner, I want a clear way to exit if something comes up, without corrupting the session data.

## Outcomes

- As a learner, I want the scenario to end with a clear outcome (safe/unsafe/late) so I know where I landed.
- As a learner, I want to know that my actions are being recorded so I can trust the debrief data afterward.

## FAA compliance

- As an operator, I want scenario play time logged as FAA-qualified time against specific topics so the system can prove the learner's 16 hours.
- As an operator, I want each run to produce an evidence packet with competencies exercised and topics covered so we can demonstrate learning to the FAA.

## Safety culture (design principle: emotional safety)

- As a learner, I want over-intervention (unnecessary takeover) to be treated as a judgment error, not a virtue, so I understand that grabbing controls prematurely is itself a problem.
- As a learner, I want a bad outcome to redirect me to the debrief (not an error page) so failure feels like a learning opportunity, not punishment.
