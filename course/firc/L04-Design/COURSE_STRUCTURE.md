# Course Structure

## FAA Requirements Overview

### Regulatory Basis

- **14 CFR SS 61.197** - Renewal of flight instructor certificate
- **14 CFR SS 61.199(a)(1)** - Reinstatement of privileges
- **AC 61-83K** - The governing advisory circular (the "bible" for FIRC design)
- **SS 61.197(b)(2)(iii)** - Legal hook: within preceding 3 calendar months, successfully complete an approved FIRC

### What the FAA Actually Requires

**13 Core Topic Areas** (not lessons, not modules -- buckets of subject matter that must be covered):

| #    | Core Topic                                              | Min Time |
| ---- | ------------------------------------------------------- | -------- |
| A.1  | Navigating in the 21st Century (GPS / TAA / automation) | ~45 min  |
| A.2  | Security-related Special Use Airspace                   | ~45 min  |
| A.3  | TSA Requirements                                        | ~45 min  |
| A.4  | Teaching / Safety Culture                               | ~45 min  |
| A.5  | Current GA Safety Trends                                | ~45 min  |
| A.6  | Pilot Deviations                                        | ~45 min  |
| A.7  | FAASTeam / WINGS                                        | ~45 min  |
| A.8  | Regulatory / Policy / Publication Updates               | ~45 min  |
| A.9  | Flight Reviews & IPCs                                   | ~45 min  |
| A.10 | Ethics & Professionalism                                | ~45 min  |
| A.11 | Loss of Control (LOC)                                   | ~45 min  |
| A.12 | ACS (Airman Certification Standards)                    | ~45 min  |
| A.13 | Accepting Student / Remote Pilot Applications           | ~45 min  |

### Time Rules

- **Total course**: >= 16 hours of "ground and/or flight instruction" (AC 61-83K SS 10.4)
- **Per core topic**: 45 minutes minimum (SS 10.5.2)
- **Elective topics**: 30 minutes minimum (SS 10.5.3) -- electives are encouraged, not required
- **Internet FIRCs**: 12 hours minimum must be lesson presentations (audio/video/text); remaining 4 hours can be testing, breaks, transitions (SS 12.1.1.3)
- See [L02-Knowledge/FAA-TIME-AND-DELIVERY.md](../L02-Knowledge/FAA-TIME-AND-DELIVERY.md) for full sourced breakdown

### Key Flexibility

- The FAA does NOT mandate number of lessons, specific lesson breakdown, or sequence
- Topics CAN be taught in combination when appropriate
- Topics CAN be interleaved and revisited
- A single scenario CAN count toward multiple core topics simultaneously
- The "13 lessons" in current FIRCs is just a provider's packaging choice

### What the FAA Cares About

1. **Coverage** - Did you teach all 13 topics?
2. **Time** - Did each get enough exposure (~45 min cumulative)?
3. **Assessment** - Did the user demonstrate learning?

### What the FAA Does NOT Care About

- Whether it's boring or ugly
- Your lesson structure or blocks
- Your game mechanics
- How fun it is

## Our Course Structure: Six Modules

We organize around experiences, not topics. Then map experiences -> topics.

### Module 1 -- Instructor Foundations & Modern Cockpit

**Time: 2.75 hours**

**FAA Core Topics Covered:**

- A.1 Navigation / GPS / TAA
- A.8 Regulatory / policy / publication updates
- A.12 ACS (Airman Certification Standards)

**Competencies Targeted:** AV-1, AV-2, AV-3, RC-4, ES-3

**Why grouped:** This is the "new baseline" for a modern CFI. GPS/automation/TAA, ACS, and current regulatory content naturally group together.

**Game Modes:**

- Glass cockpit transition scenarios
- Mode confusion challenges
- GPS failure / backup nav decisions
- ACS tagging and debriefing exercises
- "Current vs outdated guidance" challenges

**Scenario Families:**

- GPS database out of date
- Autopilot mode confusion
- RAIM / integrity issue
- Glass cockpit fixation in busy pattern
- ADS-B/weather delay misunderstanding
- TAA transition briefing
- Old lesson plan vs current requirement

### Module 2 -- Instructional Effectiveness & Safety Culture

**Time: 3.0 hours**

**FAA Core Topics Covered:**

- A.4 Teaching & Safety Culture
- A.10 Ethics & Professionalism
- A.5 Current Safety Trends in GA

**Competencies Targeted:** CJ-1, CJ-2, CJ-3, PS-1, PS-2, RM-1, RM-3

**Why grouped:** Fundamentally about instructor judgment and culture.

**Game Modes:**

- Difficult student conversations (dialogue trees)
- Pressure to sign off scenarios
- External pressure / schedule pressure
- Debriefing and intervention timing
- Accident chain reconstruction
- Risk stacking simulator

**Scenario Families:**

- Overconfident student
- Timid student
- Student who parrots but doesn't understand
- Rushed checkout with external pressure
- Owner pressures signoff
- Friend wants shortcut flight review
- Hot/high departure risk
- Weather + fatigue + get-there pressure

### Module 3 -- Loss of Control Lab

**Time: 3.25 hours**

**FAA Core Topics Covered:**

- A.11 LOC
- A.6 Pilot Deviations
- A.4 Teaching (reinforced) -- positive exchange of controls

**Competencies Targeted:** AC-1, AC-2, AC-3, OD-1, OD-2, CJ-1, CJ-2, CJ-3, RM-2

**Why grouped:** Most game-like and most obviously valuable. Heavily emphasized in AC 61-83K.

**Game Modes:**

- Base-to-final tick engine (the flagship mechanic)
- Departure stall intervention
- Runway incursion / taxi confusion
- Go-around timing decisions
- Student freeze / incorrect control response
- Replay with variation

**Scenario Families:**

- Base-to-final overshoot and stall-spin setup
- Engine failure after takeoff
- Departure stall
- Inadvertent IMC with overloaded pilot
- Complex airport taxi at night
- Readback misunderstanding
- Runway crossing confusion
- Gusty crosswind final

### Module 4 -- Airspace, Security, and Compliance

**Time: 2.0 hours**

**FAA Core Topics Covered:**

- A.2 Special Use Airspace
- A.3 TSA Requirements
- A.13 Accepting Student/Remote Pilot Applications

**Competencies Targeted:** OD-3, RC-1, RC-2, RC-3

**Why grouped:** Administrative and operational compliance duties. Important but shouldn't feel like a dead legal lecture.

**TSA Delivery Model (example of distributed instruction):**

- 15-20 min: Core instruction (FTSP system, who it applies to, when to verify, records)
- 10 min: First scenario (walk-in prospective foreign student)
- 10 min: Second scenario (transfer student, varied facts)
- 5 min: Spaced recall reinforcement
- 5 min: Recap / evidence check
- Total: ~45 identifiable minutes, feels like ~10

**Game Modes:**

- Route planning with dynamic TFRs
- Walk-in prospective student scenario beats
- IACRA / application acceptance branches
- "Stop or continue?" compliance decisions
- Document triage

**Scenario Families:**

- Cross-country route with moving TFR
- Student busts protected airspace
- Intercept recognition drill
- Non-US prospective student walk-in
- "Other school already checked me" transfer
- Missing citizenship documentation
- TSA eligibility change mid-training
- Incomplete student pilot application
- Remote pilot applicant with issue

### Module 5 -- Meaningful Evaluations

**Time: 2.5 hours**

**FAA Core Topics Covered:**

- A.9 Flight Reviews & IPCs
- A.7 FAASTeam / WINGS

**Competencies Targeted:** ES-1, ES-2, ES-3, PS-2

**Why grouped:** AC 61-83K is clear that reviews/IPCs should be meaningful and tailored, not rote checkboxes.

**Game Modes:**

- Build a tailored review plan
- Diagnose the rusty instrument pilot
- Pick the right proficiency plan
- Use WINGS as a follow-on safety tool
- Client conversation roleplay

**Scenario Families:**

- Rusty pilot who sounds polished
- Active pilot with hidden weak ADM
- Sharp pilot with hidden weak area
- Automation-heavy IFR pilot weak in fundamentals
- Friend asks for "quick signoff"
- Post-checkride proficiency plan building

### Module 6 -- Integrated Capstone

**Time: 2.5 hours**

**FAA Core Topics Covered:**

- All core topics integrated
- Replay / spaced repetition / improvement delta

**Competencies Targeted:** All domains

**Why:** Proves knowledge transfer, engagement, and improvement.

**Game Modes:**

- Mixed scenarios (multiple topics per scenario)
- Branching "day-in-the-life" CFI simulation
- Incident chain recognition
- End-of-course reflective debrief
- Final "checkride" scenarios

**Assessment Focus:**

- Comprehensive performance scoring
- Initial vs final performance comparison (improvement delta)
- Competency validation across all domains

### Total: 16.0 hours

## Topic Coverage Accounting

Each topic has a time counter. Scenarios contribute to multiple topics simultaneously.

```
TopicCoverage {
  LOC: 90 min    // Block 3 primary + Block 6 capstone
  TSA: 45 min    // Block 4 distributed + spaced reinforcement
  ACS: 45 min    // Block 1 + Block 5 debrief
  // ... etc
}
```

As long as each >= ~45 minutes and total >= 16 hours: compliant.

## Dual Time Tracking

### FAA-Qualified Time

Counts toward required topics, required coverage, FIRC completion.

### Exploratory Time

Extra scenarios, deep dives, interest-driven learning. Does NOT count toward FIRC completion.

When entering extra content:

> "This activity is outside FAA-required areas. It will not count toward your FIRC completion time."

### Completion Logic

User must have:

- > = 16 FAA-qualified hours
- All core topics covered (each >= ~45 min)
- Assessments passed
- Competency demonstrated across all required domains
