# Learning Track

You're a 30-year lapsed CFI returning to active flying and instructing. You need to relearn everything. You prefer diversity -- different topic each day or week -- over grinding one subject to completion.

## Don't Mirror Products

The products structure (categories, PRDs, frontmatter) is wrong for learning. Products are discrete buildable things. Learning domains are fluid, overlapping, and personal. A single study session might touch weather, regulations, and IFR procedures. Don't force it into boxes.

## Learning Domains

A returning CFI needs competence across all of these:

| #  | Domain                         | Notes                                                         |
| -- | ------------------------------ | ------------------------------------------------------------- |
| 1  | Regulations                    | FARs, AIM, current ADs, recent rule changes                   |
| 2  | Weather                        | METARs, TAFs, PIREPs, radar, icing, thunderstorms, decisions  |
| 3  | Airspace                       | Class A-G, special use, TFRs, ADIZ, what changed in 30 years |
| 4  | Glass Cockpits                 | G1000, GTN, modern avionics workflow, automation management   |
| 5  | IFR Procedures                 | Approaches, departures, enroute, holds, modern GPS/RNAV       |
| 6  | VFR Operations                 | Pattern work, navigation, flight following, CTAF              |
| 7  | Aerodynamics & Performance     | Refresher, weight/balance, DA/PA                              |
| 8  | Teaching Methodology           | FOI concepts, scenario-based training, learning theory        |
| 9  | ADM & Human Factors            | DECIDE, 3P, PAVE, IMSAFE, TEM, CRM, startle/surprise        |
| 10 | Safety & Accident Analysis     | NTSB cases, common errors, safety culture                     |
| 11 | Aircraft Systems               | Whatever types you'll fly                                     |
| 12 | Flight Planning & Navigation   | Modern tools, EFBs, GPS vs pilotage                           |
| 13 | Emergency Procedures           | Memory items, decision trees                                  |
| 14 | FAA Practical Standards        | ACS for CFI, what's tested, how it's tested                   |

## Tracking

Each domain gets a simple tracking file (not a full PRD) with:

- **Scope** -- what's in and out for this domain
- **Current comfort** -- 1-5 self-assessment
- **Target level** -- what "ready" looks like
- **Key resources** -- books, videos, apps, people
- **Last studied** -- date
- **Notes** -- whatever's useful

Keep these lightweight. A domain file should be under 30 lines.

## Learning Feeds Building

Content created while studying flows directly into the products as shared content. Writing flashcards for regulations -> feeds Spaced Memory Items (prd:prof:spaced-memory-items). Analyzing an NTSB report -> feeds NTSB Story (prd:aud:ntsb-story). Practicing weather decoding -> feeds WX Calls (prd:prof:wx-calls).

The products themselves -- especially prd:prof:spaced-memory-items (Spaced Memory Items), prd:prof:decision-reps (Decision Reps), and prd:prof:wx-calls (WX Calls) -- ARE the learning tools. Building them IS studying. There's no separation between "learn this material" and "build the tool that teaches it." The first user is the builder.

## Rotation

Diversity over depth. Rotate domains daily or weekly. Don't grind one domain to 5/5 before touching another. A week might look like:

- Monday: Regulations (read recent LOIs)
- Tuesday: Weather (decode 10 METARs)
- Wednesday: Glass cockpits (G1000 simulator)
- Thursday: ADM (one NTSB case study)
- Friday: IFR procedures (brief 3 approaches)

The Calibration Tracker (prd:prof:calibration-tracker) will eventually make this rotation data-driven. Until then, follow curiosity.
