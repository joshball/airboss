- [x] Diagnosed root cause: NOT vitest worker pool sharing (each worker
  process gets its own `postgres()` pool). The real cause is that
  `scripts/db/unit-test-setup.ts` runs
  `pg_terminate_backend(...) WHERE datname='airboss_unit_test'`, which
  kills every connection to that DB -- including another live run's
  worker pools -- then DROPs + reseeds it. No concurrency guard existed.

  exclusive lock (atomic `open(path,'wx')`, stale-PID reclaim).
  `vitest.globalSetup.ts` takes a machine-wide lock before provisioning
  and releases it via the returned teardown after the whole run. A
  second invocation blocks until the first finishes.
- [x] Added `scripts/lib/file-lock.test.ts` (7 tests: exclusivity,
  release, idempotent release, stale-lock reclaim, corrupt-lock reclaim,
