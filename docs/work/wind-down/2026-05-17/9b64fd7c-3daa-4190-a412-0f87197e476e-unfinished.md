## CFR body derivatives missing in fresh git worktrees

- **What**: CFR section body `.md` files under `regulations/cfr-14/<edition>/`
  and `regulations/cfr-49/<edition>/` are git-ignored (ADR 018 --
  developer-local cache derivatives). `git worktree add` only checks out
  tracked files, so a fresh worktree has zero CFR body files. The CFR seed
  adapter then skips every CFR Part ("body files not yet registered"), and
  `scripts/db/seed-references-from-manifest.test.ts` fails because CFR
  `reference` rows exist with no `reference_section` rows.
- **Where**: `libs/bc/hangar/.../seeders/cfr.ts` skip path (line ~286);
  `scripts/db/unit-test-setup.ts` seeds the unit DB; ADR 018 storage policy.
  No artifact yet -- worked around this session by `rsync`-ing the CFR
  derivatives from the parent repo into the worktree.
- **Why deferred**: not a code bug -- the suite passes in any environment
  that has run `bun run sources register cfr` (or the parent repo, which
  has the cache). It is a friction point for agents working in fresh
  worktrees who run `bun run test` there.
- **Next action**: decide whether to (a) document in `CLAUDE.md` /
  `docs/agents/` that worktree-based test runs need the CFR cache copied
  in, or (b) make the unit-test setup `rsync` / regenerate CFR derivatives
  into the worktree automatically, or (c) make
  `seed-references-from-manifest.test.ts` skip the CFR-section assertion
  when no CFR body files are present on disk (so a cache-less environment
  reports "skipped" rather than "failed"). Option (c) is the cleanest --
  it keeps the test honest where the cache exists and silent where it
  legitimately cannot run.
- **Severity**: low
