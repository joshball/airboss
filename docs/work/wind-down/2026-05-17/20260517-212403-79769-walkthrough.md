## Integration sweep -- reworked dispatcher

**Command**: `bun run test integration [subcommand] [flags]`

**How to test manually** (run from a worktree or clean checkout):

1. `bun run test integration --help` -- prints our own help (not Playwright's
   passthrough).
2. `bun run test integration list` -- compact per-book plan, tree-grouped under
   `kind/`, runs zero tests.
3. `bun run test integration` (default) -- fast sanity-tier sample (~12
   URLs/book). Live per-book progress dashboard repaints in place on a TTY.
4. `bun run test integration --book handbook/phak` -- runs only that book.
   First run builds flightbag (cached after via
   `apps/flightbag/build/.sweep-buildhash`).
5. `bun run test integration --full` -- exhaustive ~5500-URL sweep.
6. `bun run test integration --resume` -- re-runs only URLs that failed or
   never ran last time (from `tests/integration/.out/last-run.json`).

**What to look for**:

- Output is the custom reporter only -- grouped per-book failures, per-tier
  counts + timings, what-was-tested summary, per-book retest commands.
- On a crash/interrupt: a `RUN DID NOT FINISH` banner, not a wall of fake
  failures.
- Artefacts under `tests/integration/.out/`: `coverage-report.md`,
  `coverage-failures.txt`, `coverage-summary.json`, `manifest.json`,
  `progress.ndjson`, `last-run.json`.
