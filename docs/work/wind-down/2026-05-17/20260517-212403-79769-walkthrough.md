# Walkthrough -- integration coverage sweep rework

This session reworked the `bun run test integration` flightbag coverage sweep:
output, sampling, resume, server stability. The artifacts are CLI commands, not
web pages.

### Integration sweep -- new dispatcher commands

**Command**: `bun run test integration [subcommand] [flags]`

**How to test manually** (once Track A is merged -- currently on the
`track-a/*` branch, not yet on main):

1. `bun run test integration --help` -- our own help text renders, not Playwright's
   passthrough help.
2. `bun run test integration list` -- prints the compact per-book plan
   (tree-grouped under `kind/`), runs zero tests, no per-title spam.
3. `bun run test integration --book handbook/tips-mountain-flying` -- runs only
   that book. First run builds flightbag (~20-40s); second run uses the cached
   build (`apps/flightbag/build/.sweep-buildhash`).
4. `bun run test integration` (default) -- sanity-tier sample (~12 URLs/book),
   fast. Live per-book progress dashboard repaints in place on a TTY.
5. `bun run test integration --full` -- the exhaustive ~5500-URL sweep.
6. `bun run test integration --resume` -- re-runs only URLs that failed or never
   ran in the previous run (from `tests/integration/.out/last-run.json`).

**What to look for**:

- The reporter is the only terminal output -- grouped per-book failures, per-tier
  counts + timings, a what-was-tested summary.
- On a crash/interrupt: a loud `RUN DID NOT FINISH` banner, not a wall of fake
  failures.
- Artefacts under `tests/integration/.out/`: `coverage-report.md`,
  `coverage-failures.txt`, `coverage-summary.json`, `manifest.json`,
  `progress.ndjson`, `last-run.json`.

**Known caveats**:

- The production server (`bun ./apps/flightbag/build/index.js`) does NOT boot on
  current main: `z.coerce.boolean(...).meta is not a function` -- a repo-wide
  zod/better-auth version mismatch that also breaks `apps/study`'s build. Until
  that is fixed, the sweep's prod-build server is not runnable end-to-end.
- Track A's webServer command must use `bun ./build/index.js`, not `node build`
  (verify on the `track-a/*` branch before merging).
