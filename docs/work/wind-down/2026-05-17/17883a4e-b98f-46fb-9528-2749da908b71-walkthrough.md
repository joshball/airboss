# Walkthrough -- flightbag reader test coverage + route bug fixes

This session shipped three PRs: an e2e test-determinism fix, a new integration
coverage suite for the flightbag reader, and fixes for the two reader-route
404 bugs that suite surfaced. All user-facing artifacts are flightbag reader
pages and the new `bun run test integration` command.

The flightbag app runs at `https://flightbag.airboss.test` with `bun run dev`
(dev port 9640). The integration suite runs against its own isolated
DB (`airboss_integration`) and port (9647).

## Integration coverage suite (PR #999)

**Command**: `bun run test integration`

**How to test manually**:

1. Ensure the local Postgres container (`airboss-db`, port 5435) is running.
2. Run `bun run test integration` from the repo root.
3. The Playwright `globalSetup` provisions `airboss_integration` (drop +
   create + pg_trgm + schema push + full seed), then boots one flightbag
   vite process on port 9647.
4. 32 workers sweep every flightbag reader URL derived from the seeded
   reference registry: ~3400 tests across three tiers (sanity / structural /
   content).

**What to look for**: All tests pass (3408/3408 after PR #1005). The custom
reporter writes `tests/integration/.out/coverage-report.md` -- a per-tier
pass/fail breakdown plus every skipped row with its reason.

**Known caveats**: The suite reuses the e2e `globalSetup` (now provisions
`airboss_e2e` + `airboss_integration` in parallel). `PLAYWRIGHT_SKIP_DB_SETUP=1`
skips the provisioning for fast re-runs against an already-seeded DB.

## AC appendix chapter pages (PR #1005)

**URL**: `https://flightbag.airboss.test/ac/61-65/j/appendix-a`

**How to test manually**:

1. `bun run dev` (flightbag on 9640).
2. Open the URL above (Advisory Circular 61-65J, Appendix A).
3. Before PR #1005 this 404'd; the chapter route's `CHAPTER_SHAPE` regex
   rejected the hyphen in `appendix-a`.

**What to look for**: HTTP 200, the page renders the appendix content, and
the heading reads "Appendix A" (not "Chapter appendix-a"). 23 AC appendix
URLs across 5 ACs (120-71, 25-7, 61-65, 61-98, 90-66) are now reachable.

## Handbook front-matter pages (PR #1005)

**URL**: `https://flightbag.airboss.test/handbook/avwx/8083-28B/front-matter/0.2`

**How to test manually**:

1. `bun run dev` (flightbag on 9640).
2. Open the URL above (Aviation Weather Handbook, Preface).
3. Also try `/front-matter/0.1` (Cover) and `/front-matter/0.3`
   (Acknowledgments). Before PR #1005 these had no reader route at all.
4. Open a handbook landing page (e.g. `/handbook/avwx/8083-28B`) and confirm
   the rail TOC lists the front-matter entries with working links.

**What to look for**: HTTP 200, the front-matter body content renders with a
non-empty H1, and the TOC drawer / prev-next nav route front-matter rows to
the new `/front-matter/[code]` leaf route. 36 handbook front-matter URLs
across 6 handbooks are now reachable.

**Known caveats**: The route param `[code]` carries the full row code (`0.2`,
with the dot). SvelteKit dynamic params handle the dot fine; verified in a
real Chromium load.

## e2e representative-pages spec (PR #981)

**Command**: `bun run test e2e` (the `flightbag representative pages` specs)

**How to test manually**:

1. Run `bun run test e2e` (or scope to the spec:
   `bunx playwright test tests/e2e/flightbag/representative-pages.spec.ts`).

**What to look for**: No "Test not found in the worker process" failures. The
spec generates parameterised tests from a DB query; PR #981 added an `asc(code)`
tiebreaker to the ORDER BY so every Playwright worker generates identical test
names. 93/93 pass.
