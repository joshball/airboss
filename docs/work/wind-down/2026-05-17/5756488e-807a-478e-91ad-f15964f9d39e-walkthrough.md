# Walkthrough -- zod 4 alignment, integration dispatcher fix, branch triage

This session was infra/dependency work plus a branch-cleanup triage. The
user-testable artifacts are CLI commands, built servers, and one theme change.

### Production-build boot (the fixed zod crash)

**Command**:

```bash
set -a && source .env && set +a
(cd apps/flightbag && bunx vite build && PORT=9701 bun ./build/index.js)
(cd apps/study    && bunx vite build && PORT=9702 bun ./build/index.js)
```

**How to test manually**:

1. Build either app with `bunx vite build` from the app dir (needs `DATABASE_URL`).
2. Boot the built server: `bun ./build/index.js`.
3. Before PR #1064 this crashed at boot with
   `TypeError: z.coerce.boolean().meta is not a function`.

**What to look for**: server prints `Listening on http://0.0.0.0:<port>`;
`curl` returns 200 (flightbag) / 303 (study auth redirect). No zod TypeError.

**Known caveats**: a fresh worktree has no `.env` -- copy it from the parent
repo. Build needs `DATABASE_URL` for the SvelteKit postbuild analyse step.

### Integration sweep dispatcher

**Command**: `bun run test integration [list|--book <name>|--full] [flags]`

**How to test manually**:

1. `bun run test integration --help` -- friendly help renders.
2. `bun run test integration list` -- compact per-book tree plan, **exit 0**
   (PR #1065 fixed this; it used to exit 1 on a successful listing).
3. `bun run test integration --book handbook/tips-mountain-flying` -- runs that
   one book (12/12 pass), builds flightbag once, caches on the 2nd run.
4. `bun run test integration` -- default sample sweep, 209/209 pass, every book,
   live progress dashboard, no server OOM.

**What to look for**: grouped per-book output, no per-test spam, artefacts under
`tests/integration/.out/` (manifest.json, progress.ndjson, last-run.json,
coverage-report.md, coverage-failures.txt, coverage-summary.json).

**Known caveats**: `bun run test integration --full` (~3484 URLs) has NOT had a
clean verified pass -- see the unfinished file. Orphan webServer processes on
port 9647 make a run abort with 0 URLs checked; kill stale `build/index.js` /
`playwright` procs before a full run.

### Theme: deepInk contrast tokens (PR #1067)

**Where**: any status pill / banner / toast / badge across the apps -- e.g.
`http://localhost:5173/` study app status chips, hangar ingest-review badges.

**How to test manually**:

1. Run any app (`bun run dev`).
2. Look at status pills / wash-background banners that previously used
   `--signal-X-ink`. They now use `--signal-X-deep-ink` -- a darker text-on-wash
   variant.
3. The contrast-matrix test (`libs/themes`) went from 81 passed / 38 skipped to
   119 passed / 0 skipped.

**What to look for**: status text on tinted backgrounds is legibly dark (light
themes) / light (dark themes); no washed-out low-contrast text.
