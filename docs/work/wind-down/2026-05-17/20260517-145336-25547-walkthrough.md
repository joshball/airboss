## 3. Pre-commit guard against commits on main

**Command**: a normal `git commit` while on the `main` branch.

**How to test manually**:

1. On `main`: `git commit --allow-empty -m test` -- the commit is refused with a message
   pointing to `git checkout -b`.
2. Override path: `AIRBOSS_ALLOW_MAIN_COMMIT=1 git commit --allow-empty -m test` -- allowed.
3. On any feature branch: commits work normally.
4. `bun scripts/setup.ts` -- the new "install git hooks" step copies
   `scripts/hooks/pre-commit` into `.git/hooks/pre-commit`.

**What to look for**: the hook blocks `main` / `master` only; the escape hatch works;
feature branches are unaffected. Worktrees share the parent repo's hooks, so sub-agent
worktrees are covered too.

**Known caveats**: the hook lives in `.git/hooks/` (not version-controlled). The tracked
source is `scripts/hooks/pre-commit`; `setup.ts` installs it. A fresh clone needs
`bun scripts/setup.ts` run once before the guard is active.
