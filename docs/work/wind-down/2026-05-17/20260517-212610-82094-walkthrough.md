# Walkthrough -- e2e + md-format tooling session

This session was tooling and test infrastructure -- no new user-facing
pages or routes. The artifacts to verify are CLI commands and the
scheduled-job registration.

### e2e suite runs end-to-end

**Command**: `bun run test e2e`

**How to test manually**:

1. From the repo root, run `bun run test e2e`.
2. The DB setup (`e2e-setup.ts`) should complete the full seed -- the AC
   00-45H seed-mapping fix (PR #1034) was the blocker; it now resolves.
3. The suite should report all specs passing -- the 6 spec failures
   fixed in PR #1040 (practice-wx-drill/mastery/replay, reference-wx-
   examples, unauthed/auth, content-census) are green.

**What to look for**: `seed: done`, then Playwright reports 0 failed.

**Known caveats**: e2e boots 4 vite webServers; if a prior run left a
process on ports 9600-9603 / 9643 / 9647, kill it first.

### md-format is idempotent and scoped

**Command**: `bun tools/md-format/bin.ts --check --all`

**How to test manually**:

1. Run the command -- it should report `clean (N file(s) checked)`.
2. Run `bun tools/md-format/bin.ts --all` then re-run `--check --all` --
   still clean (idempotent: format-once converges).
3. Confirm it does NOT touch ingested FAA corpora: `git status` after
   `--all` shows no changes under `handbooks/`, `ac/`, `aim/`, `safo/`,
   `info/`.

**What to look for**: `clean`, and no FAA-corpus files in `git status`.

### markdown-format-drift scheduled job

**Command**: `scripts/scheduler/list.sh`

**How to test manually**:

1. Run `scripts/scheduler/list.sh` -- `markdown-format-drift` should be
   listed, `enabled yes`, schedule `0 9 1 * *`.
2. `launchctl list | grep markdown-format-drift` -- confirms it is loaded
   into launchd.
3. Optionally dry-run the work: `bash scripts/scheduled-jobs/markdown-format-drift/run.sh`
   -- exits 0, reports drift (if any) but does not commit (the wrapper
   handles PR mechanics on the real scheduled fire).

**What to look for**: job listed + loaded; `run.sh` exits 0.

**Known caveats**: launchd jobs only fire while the laptop is awake;
`scripts/scheduler/.scheduler-config` is per-machine (git-ignored) and
had to be reconstructed this session.
