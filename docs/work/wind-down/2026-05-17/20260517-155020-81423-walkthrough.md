# Walkthrough: Navigating FAA Documentation course

This session authored and built the `faa-documentation-navigation` course. The artifacts are
file-backed course content (a course definition, knowledge-graph nodes, a reference set, and
glossary entries) consumed by the existing course and knowledge seed pipelines. There is no
new app route or endpoint. Manual verification is a content walk plus the seed validators.

## Course: Navigating FAA Documentation

**What it is**: A standalone orientation course teaching how to navigate the FAA documentation
ecosystem (the document families, the AC numbering scheme, citation anatomy, supersession, and
cross-reference triangulation).

**Files to review**:

- Course definition: `course/courses/faa-documentation-navigation/manifest.yaml` + 6 section
  files under `sections/` (`s1-why-the-ecosystem` through `s6-finding-things-and-cross-reference`).
- Knowledge nodes: `course/knowledge/regulations/{faa-document-ecosystem,ac-numbering-system,faa-citation-anatomy,faa-cross-reference-triangulation}/node.md`.
- Reference artifact: `course/regulations/references/document-families/` (12 family `page.md`
  files + `README.md` index).
- Glossary: 13 entries in `libs/help/src/glossary/entries.ts` with bodies under
  `libs/help/src/glossary/content/`.

**How to test manually**:

1. Read `course/courses/faa-documentation-navigation/manifest.yaml` and walk the 6 sections in
   order. Confirm each section frames its topic discovery-first and hands off to a knowledge node.
2. Read the 4 knowledge nodes. Confirm the 7-phase discovery structure (context / problem /
   discover / reveal / practice / connect / verify) and that the `discover` phase asks the
   learner to reason before the `reveal`.
3. Read `course/regulations/references/document-families/README.md` (the browsable family map)
   and spot-check several `page.md` files for the binding-vs-advisory call.
4. In the study app, hover any of the 13 new glossary terms (CFR, AC, AIM, ACS, AD, NOTAM, ...)
   to confirm the tooltip `short` text renders.
5. Run the seed validators: `bun run db build` dry-run path (knowledge + course) and
   `bun test libs/help/src/glossary/index.test.ts` (glossary loader).

**What to look for**: The regulatory-vs-advisory distinction taught explicitly and consistently
across course, nodes, and reference pages. The AC-numbering discovery exercise uses one stable
five-number set (`AC 61-65`, `AC 91-73`, `AC 20-138`, `AC 150-5300-13`, `AC 00-46`) identical in
the section, the node, and the recall cards.

**Known caveats**:

- The manual test plan (`docs/work-packages/faa-documentation-navigation/test-plan.md`, 13 FDN
  scenarios) has not been hand-walked by the user. Per the project rule, the course is not
  "feature complete" until the user signs off; the WP spec stays at `status: draft`.
- The course content is on main and all seed validators pass, so it is safe to load and review.
