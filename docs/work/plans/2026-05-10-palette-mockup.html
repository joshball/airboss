Key behaviors illustrated

  • Hoist banner. Single tier-1 match -> banner -> Enter opens it directly.
  • Synonym chip. "wx == weather" -- visible feedback that the query was expanded.
  • Parent grouping. Inside FAA Resources, sections cluster under their parent doc with a count. Avoids "20 PHAK chapters drowning out everything else."
  • Provenance in meta. Every row shows doc code / corpus / chapter so the user knows what they're looking at without hover.
  • Match highlighting. Matched tokens in titles get amber highlight.
  • Color coding by type. Amber = FAA. Violet = knowledge / glossary. Cyan = airboss course. Green = user's own stuff. Rose = external. Muted = command.
  • Counts & show-all. Each column header shows "N of M -- show all" linking to a filtered list view. No infinite scroll inside the palette.
  • Filters footer. Suggests doc: / kind: / mine when the result set is large.

Doc-picker behavior (Variant D)

  • Detected as doc-code: FAA-H-, FAA-S-, AC , 14 CFR , 49 CFR , Part , §, bare digits like 8083 or 61-, or known abbreviations (AvWX, PHAK, AFH, IFH, IPH, RMH, AIH).
  • Right pane becomes doc list. Detail pane re-purposes as a sorted list of matching FAA docs, showing code + title + abbrev. Highlighted row matches the cursor.
  • Enter opens the doc. Cmd+Enter sets doc:<code> as a filter chip and returns to normal search-inside-doc mode.
  • Left side shows recents + actions + filters. So you can still keyboard-navigate even while the doc list is the focus.
  • Numeric prefix sort. 808 orders 8083-2 → 8083-3 → 8083-9 → 8083-15 → 8083-16 → 8083-25 → 8083-28 (numeric, not alpha-string).
