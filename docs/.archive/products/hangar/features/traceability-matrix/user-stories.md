# User Stories: Traceability Matrix Editor

## Content Author

- As a content author, I want to see all 13 FAA topics in one view so I can understand what's covered and what's missing.
- As a content author, I want to see the exact FAA title for each topic (from AC 61-83K) so I know precisely what the FAA expects.
- As a content author, I want to link scenarios, modules, and competencies to each topic so the matrix reflects our actual curriculum.
- As a content author, I want the matrix to auto-populate from existing content so I don't manually enter what the system already knows.
- As a content author, I want to select structured assessment methods per topic (scenario, knowledge-check, debrief, practical) so the FAA package exports consistently.
- As a content author, I want to see estimated time per topic computed from my scenarios so I can verify we meet the 45-minute minimum.
- As a content author, I want deleted scenarios flagged in the matrix so I understand why coverage changed.

## Compliance Manager

- As a compliance manager, I want to see pass/fail status per topic so I can quickly find gaps.
- As a compliance manager, I want to sync the matrix from current content so it stays up to date.
- As a compliance manager, I want the matrix to feed into the FAA package generator so I don't recreate it.

## FAA Reviewer

- As an FAA reviewer reading the exported matrix, I want to see which scenarios, modules, and competencies map to each topic so I can verify coverage without asking the provider.
