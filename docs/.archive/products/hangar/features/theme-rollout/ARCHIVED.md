# Archived: Theme Family Rollout (2026-04-25)

This feature package described a families-based architecture (`workbench` / `focus` / `brand`) that was never implemented. It was marked `status: done` because Phase 1 ("lock the contract") finished as a paper exercise, but Phases 2-6 (shells, families, governance) were superseded before any of them shipped.

## What actually shipped instead

The eight-package [Theme System Overhaul](../../../../../work-packages/theme-system-overhaul/README.md) (PRs #78 - #85, #105, #114, #183, #190). Three orthogonal axes (`theme` × `appearance` × `layout`) replaced the families concept entirely.

## Why this was archived (not deleted)

The Phase 1 decisions captured here (font-role count, file structure, glass-cockpit/aviation handling) informed the overhaul that followed, even though the architectural framing was rejected. Kept for historical context.
