# The Option A overhaul (2026-04)

How the theme system shipped end-to-end across 8 PRs in ~48 hours. This doc is the "how did we actually do it" companion to [02-ARCHITECTURE.md](02-ARCHITECTURE.md)'s "what is it".

## Goal

Migrate from a 100+ `--ab-*` primitive vocabulary to the role-token architecture in [02-ARCHITECTURE.md](02-ARCHITECTURE.md). Zero hex/rgb/px/ms literals in components. Zero ranked names (`primary`/`secondary`). Every theme x appearance combination clears WCAG AA. User-visible dark/light/system appearance toggle. Enforcement shipped with the architecture, not after.

## The failed first attempt

PR #69 tried to land it all at once: contract + emit + role-token vocabulary + migration of 2,614 call sites across 65 files. `main` moved during the week it took to land. The PR accumulated 10 merge conflicts, some hiding main-side bug fixes that would have been silently reverted if we'd resolved them naively. Closed #69 and restarted with a two-wave strategy.

## The two-wave strategy

Split the work so each wave is atomic:

1. **Land the foundation behind a compat-alias shim** (#78 - #82). Role tokens exist, primitives use them, but every legacy `--ab-*` name is re-emitted as a CSS custom property pointing at a role-token value. Unmigrated pages render identically.
2. **Sweep the call sites + delete the shim** (#83 - #85). App-by-app codemod pass. Once all apps are on role tokens, delete `legacy-aliases.ts` entirely.

Both waves keep `main` in a shippable state every step.

## The wave graph

```text
#78 foundation (role tokens + emit pipeline + compat-alias shim + 12 primitive migration)
  |
  v
#79 contract extension (typed placeholders for typography/control/sim)
  |
  |-- #80 typography packs ---\
  |-- #82 enforcement --------|-- all mergeable in parallel; landed in completion order
  |-- #81 UI primitives ------/
  |
  |-- #83 study sweep (codemod + delete study-only aliases) ---\
  |-- #84 sim theme + migration -------------------------------|-- parallel
  |                                                            /
  v
#85 dark palettes + appearance toggle + hangar migration + retire legacy aliases
```

Eight PRs. Four waves. Two of the four waves ran parallel agents.

## Why the contract extension (#79) before Wave 2

Waves 2 and 3 put multiple agents in flight simultaneously. Agents A (typography), B (enforcement), and C (primitives) would all have touched `libs/themes/contract.ts` if it didn't already have the typed slots they needed. `#79` landed the typed placeholders for `TypographyPack`, `ControlTokens`, and `SimTokens` so each agent only filled its own slot. No one restructured the contract under anyone else's feet.

This is the load-bearing rule for parallel agents: **scope by file, not by block**. If two agents share a file, they share a block, and blocks collide even when the edits are semantically orthogonal.

## File-ownership scoping (the parallel rule)

Every Wave 2/3 agent was briefed with explicit file ownership. Example from Wave 2:

| Agent | Owns                                                | Does NOT touch                   |
| ----- | --------------------------------------------------- | -------------------------------- |
| A #2  | `libs/themes/core/defaults/**/typography.ts`, packs | `scripts/`, primitives, control.ts |
| B #3  | `scripts/lint-tokens.ts`, `scripts/codemod-tokens.ts`, `tools/theme-*/`, contrast tests | typography, control.ts, primitives |
| C #4  | `libs/ui/src/components/*.svelte`, `control.ts`     | typography, scripts, contract structure |

One collision appeared at merge time: #81's new primitives (Dialog/Table/Spinner) introduced token names that #82's lint rule didn't know about. Fixed in rebase by extending the lint rule's `KNOWN_PREFIXES` list. The fix was local and took two minutes because the collision was in a single prefix-list, not an entangled block.

## The compat-alias bridge

Key design decision. `libs/themes/legacy-aliases.ts` (existed #78 through #84, deleted in #85) shipped one CSS custom property per legacy `--ab-*` name, each pointing at the new role-token value:

```css
--ab-color-primary: var(--action-default);
--ab-space-md:      var(--space-md);
--ab-font-size-base:var(--type-reading-body-size);
```

This turned "migrate 2,614 call sites" into an independent optimization problem: the app renders correctly the whole time; the sweep can land one folder at a time, one app at a time.

The file was emitted into `generated/tokens.css` alongside the real role tokens, so the shim survived the full build pipeline. When #85 deleted the shim, it also deleted:

- The import chain (`legacy-aliases.ts` -> `emit.ts` -> `generated/tokens.css`)
- The `LEGACY_ALIAS_MAP` consumed by `tools/theme-codemod`
- The legacy-name allowlist in `tools/theme-lint`
- The `legacy-alias-coverage.test.ts` that enforced "every `--ab-*` used has an alias entry"

Compat layers are scaffolding. They come out.

## Enforcement stack

Shipped in #82, not deferred. Three tools:

### theme-lint

`tools/theme-lint/bin.ts` scans `.svelte` `<style>` blocks + `.css` files. Flags:

- Hex / `rgb()` / `rgba()` / `hsl()` / `oklch()` literals on color-ish properties
- Named CSS colors (`red`, `white`) on color-ish properties
- Raw `px`/`rem`/`em` on blocklisted properties where tokens exist (`0` and `1px` are exempt)
- Raw `ms`/`s` durations
- Literal `font-family` strings
- `var(--unknown-token)` - cross-checks against `vocab.ts`, `legacy-aliases.ts` (while it existed), and emitted prefix families (`--type-*`, `--button-*`, `--input-*`, `--sim-*`, etc.)

Inline exception: `/* lint-disable-token-enforcement: <reason> */`. Grandfather list at `tools/theme-lint/ignore.txt` for cases that want a timeline rather than a permanent exception. Every migration PR shrinks the ignore list.

Wired into `bun run check` so CI fails on new violations.

### theme-codemod

`tools/theme-codemod/bin.ts` mechanically rewrites:

- `--ab-*` names -> role-token equivalents via the legacy-alias map (file now deleted; codemod stripped of this pass)
- Raw radius literals (`4px`, `8px`) -> nearest `var(--radius-*)` rung
- Known motion durations (`120ms`, `200ms`) -> `var(--motion-fast|normal)`
- Known font stacks -> `var(--font-family-*)`
- Color literals flagged with `/* TODO-theme */` comments (ambiguous; human-reviewed)

Idempotent. Auto-refreshes the lint ignore file after a run.

Used in-band by `#83` (study sweep) and `#84` (sim sweep).

### Contrast matrix

`libs/themes/__tests__/contrast-matrix.test.ts` computes WCAG AA/AAA ratios for every (theme, appearance, role-pair). 11 required pairs per combination (body ink on page surface, action ink on action base, signal ink on signal wash, etc.). Failures are hard test failures. Six advisory pairs (edge visibility, signal ink on wash in light) log ratios without failing, pending the dark-palette pass.

Enforces the contract: no role token can land values that break readability under AA.

## Dark palettes + appearance UX (#85)

Final package. Three deliverables:

1. **Real hex dark palettes** for `airboss/default`, `study/sectional`, `study/flightdeck`. Stayed in hex (not OKLCH) so the contrast matrix kept enforcing; the matrix skips OKLCH pending a sRGB converter. `sim/glass` was already dark-only from #7.
2. **Appearance toggle**: `APPEARANCE_COOKIE` + `parseAppearancePreference` in `@ab/themes`, `hooks.server.ts` reads the cookie into `locals.appearance`, `(app)/+layout.server.ts` returns it, `(app)/+layout.svelte` reactive mirror, three-option toggle (system/light/dark) in the identity menu, `POST /appearance` endpoint, pre-hydration script honors `system` via `prefers-color-scheme`.
3. **Hangar migration + compat-alias deletion**: ran the codemod on `apps/hangar/src`, removed the last grandfather-ignore entries, deleted `libs/themes/legacy-aliases.ts` and everything that imported it. Generated `tokens.css` dropped from ~150KB to ~110KB.

## Results

- Zero `--ab-*` references anywhere in the repo.
- `bun run check` clean: 0 errors, 0 warnings.
- `bun run themes:emit` byte-deterministic across consecutive runs (hashed in CI).
- Contrast matrix AA-passes every (theme, appearance).
- User appearance preference persists across refresh, live-updates under system change, respects `/sim*` forced-dark.
- 8 primitives added in #4; all 20 primitives consume role tokens.
- Compat layer deleted within 7 days of being introduced.

## What to copy for the next overhaul

- **Two-wave bridge strategy** whenever the sweep exceeds one-PR-worth of drift risk.
- **File-ownership scoping** for multi-agent parallel work. Briefs must explicitly list "owns" and "does NOT touch" paths.
- **Contract extension as a pre-wave step** when N parallel agents would otherwise touch shared type files.
- **Compat layer with an expiration date.** Ship it knowing which PR deletes it. If you can't name that PR, don't ship the compat layer.
- **Enforcement lands before the migration sweep.** Lint + codemod in #82 meant #83/#84/#85 had automated tooling instead of manual rewrites.
- **Committed generated CSS** with deterministic output. Reviewers see the cascade every PR.

## What not to copy

- Do not attempt atomic multi-thousand-site sweeps. #69 proved the failure mode.
- Do not let a compat layer outlive its sweep package. Delete it in the same release train.
- Do not skip the contract-extension pre-wave when parallelizing. #69 started with a planning conversation where agents kept colliding on `contract.ts`; #79 solved it in one PR.
- Do not defer contrast tests. #82 landing before #85 caught three contrast misses during the dark-palette design.
