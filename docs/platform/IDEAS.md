# Ideas

Ideas, approaches, and insights that might shape the course or platform. This is the intake funnel -- everything lands here first. Good ideas get promoted to [DESIGN_PRINCIPLES.md](DESIGN_PRINCIPLES.md) or feature specs. Bad ideas get archived with a reason.

## How This Works

1. **Capture immediately.** When an idea comes up (conversation, research, testing, feedback), add it here. Don't evaluate yet.
2. **Review regularly.** Every 2 weeks (or at the start of each feature cycle), review this list:
   - Promote strong ideas to DESIGN_PRINCIPLES.md (if they're principles) or to a feature spec (if they're features)
   - Archive rejected ideas to the bottom with a reason (never delete -- the reason is valuable)
   - Leave undecided ideas for next review
3. **Tag with source.** Where did this idea come from? Research, user feedback, competitor analysis, design session, implementation insight?
4. **Check implementation.** When completing a feature, scan this list: did we use the ideas we meant to? Did we miss any that apply?

### Engagement & Time Quality

| Idea                                                  | Source              | Notes                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             | Status |
| ----------------------------------------------------- | ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| Active interaction tracking vs seat time              | Playtest 2026-04-07 | The FAA requires 16 hours of "instruction" (AC 61-83K SS 10.2). Traditional FIRCs: maybe 50% of that time is real engagement, and of that 50%, maybe half is actual thinking. So ~4 hours of real learning in 16 hours. With active interaction (scenarios, decisions, debriefs), engagement is near 100% when you're doing it. Track both: "FAA-qualified time" (for compliance) and "active engagement time" (actual clicks, decisions, reading). Show the difference. Maybe the FAA would accept 8 hours of fully engaged time over 16 hours of slide-reading. | New    |
| Pause on inactivity                                   | Playtest 2026-04-07 | FAA time should pause when the learner is inactive (no clicks, no scrolling, no input for N seconds). Current system tracks duration start-to-finish. Need: activity heartbeat, pause threshold, resume detection. This is both an integrity measure and an honest time-tracking mechanism.                                                                                                                                                                                                                                                                       | New    |
| Learner-directed deep dives within topic areas        | Playtest 2026-04-07 | What if the instructor wants to spend an hour reading and studying regulations? Or deep-dive into lesson planning? Currently can't do that in a static course. Allow learners to choose study areas within the 13 FAA topics. Time reading the CFI Handbook should count. Time studying the AIM should count. Give them a library of resources per topic and track time spent. Some people learn by reading, some by doing -- support both.                                                                                                                       | New    |
| MSFS integration for visualization                    | Playtest 2026-04-07 | Use Microsoft Flight Simulator as a visual tool. Show what an approach looks like. Record what a stall looks like. Put an approach in and watch it play out. Use it as a teaching aid, not a game. Link sim replays to scenario debriefs.                                                                                                                                                                                                                                                                                                                         | Future |
| Recorded instructor conversations as learning content | Playtest 2026-04-07 | Record a real CFI teaching a student to do stalls. Transcribe it. Play the audio/video. Have the learner analyze the conversation: what did the instructor do well? What would you do differently? Link the conversation to the sim replay. This is how real debriefs work -- review the tape.                                                                                                                                                                                                                                                                    | Future |
| Engagement quality metrics                            | Playtest 2026-04-07 | Track and display "official" FAA compliance time vs "actual engagement" time. Decision density (decisions per minute), reading depth (scroll depth, time on reference pages), question accuracy trends. Show learners their own engagement quality. Explore whether the FAA would accept a shorter fully-engaged course as equivalent to a longer passive one -- this could be a competitive advantage and a regulatory innovation.                                                                                                                               | New    |
| Multiple learning modalities per topic                | Playtest 2026-04-07 | For each FAA topic, offer multiple ways to learn: scenarios (active judgment), reading (reference library), questions (knowledge check), discussion (recorded conversations), visualization (MSFS or diagrams). Let the learner choose their path. Track time across all modalities toward the topic's 45-minute minimum. Different people learn differently -- a single delivery mode is a limitation, not a feature.                                                                                                                                            | New    |
